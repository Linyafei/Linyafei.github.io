Git 常用命令
2016-09-23

git config 的工具，专门用来配置或读取相应的工作环境变量。
git 配置用户信息
git config –system //系统级
git config –global //用户级
git config //当前文件级
git config –global user.name “linyafei”
git config –global user.email “linyafeiy@gmail.com“
文本编辑器
git config –global core.editor emacs/Vim/Vi
差异分析工具
git config –global merge.tool vimdiff

查看配置信息
git config –list
vim ~/.gitconfig
git config user.name

查看帮助
git help
git help pull
git pull –help
man git -pull

创建仓库
git init

命令用于查看项目的当前状态。查看在你上次提交之后是否有修改。
git status
git status -s 获得简短的结果输出
AM 状态的意思是，这个文件在我们将它添加到缓存之后又有改动

执行 git diff 来查看执行 git status 的结果的详细信息。
git diff 尚未缓存的改动
git diff –cached 查看已缓存的改动
git diff HEAD. 查看已缓存的与未缓存的所有改动
git diff –stat 显示摘要而非整个 diff

命令可将文件添加到缓存
git add .
git add test.txt

将缓存区内容添加到仓库中
git commit -m “add test.txt”
git commit -am “xxx” add 和 commit 命令合并使用

暂存区的目录树会被重写，被 master 分支指向的目录树所替换，但是工作区不受影响。
git reset HEAD 命令用于取消已缓存的内容。
倒数第二次提交
git reset –hard HEAD
命令用于取消已缓存的内容
git reset HEAD
git reset HEAD <文件>

git rm 会将文件从缓存区和你的硬盘中（工作目录）删除。
会直接从暂存区删除文件，工作区则不做出改变
git rm –cached 不从工作区中删除文件

要从Git中移除某个文件，就必须要从已跟踪文件清单中移除，然后提交
git rm
如果删除之前修改过并且已经放到暂存区域的话，则必须要用强制删除选项 -f
git rm -f
如果把文件从暂存区域移除，但仍然希望保留在当前工作目录中，换句话说，仅是从跟踪清单中删除，使用 –cached 选项即可
git rm –cached

可以递归删除，即如果后面跟的是一个目录做为参数，则会递归删除整个目录中的所有子目录和文件：
git rm –r *

命令时，会用暂存区全部或指定的文件替换工作区的文件。这个操作很危险，会清除工作区中未添加到暂存区的改动。
git checkout . 或者 git checkout –

命令时，会用 HEAD 指向的 master 分支中的全部或者部分文件替换暂存区和以及工作区中的文件。这个命令也是极具危险
性的，因为不但会清除工作区中未提交的改动，也会清除暂存区中未提交的改动。
git checkout HEAD .” 或者 “git checkout HEAD “

命令用于移动或重命名一个文件、目录、软连接。
git mv
git mv readme.txt readme2.txt

git clone
git clone <仓库名> <新路径>
touch .gitignore 在忽略文件里的文件不回添加到仓库中

前边有 * 代表是当前分支
无参数显示所有分支
git branch
创建分支
git branch (branchname)

切换分支命令:
git checkout (branchname)
创建新分支并立即切换到该分支下，从而在该分支中操作。
git checkout -b (branchname)

删除分支命令：
git branch -d (branchname)

合并分支命令:
git merge

列出历史提交记录
git log
查看历史记录的简洁版本
git log –oneline
查看历史中什么时候出现了分支、合并
git log –graph
逆向显示所有日志
git log –reverse –oneline
查找指定用户的提交日志
git log –author=lyf –oneline -5

如果你要指定日期，可以执行几个选项：–since 和 –before，但是你也可以用 –until 和 –after
git log –oneline –before={3.weeks.ago} –after={2010-04-18}

标签信息
git log –decorate
git log –oneline –decorate –graph

查看标签
git tag
添加标签
git tag -a v1.0
指定标签信息命令：
git tag -a -m “runoob.com标签”
PGP签名标签命令：
git tag -s -m “runoob.com标签”

查看当前的远程库
git remote
git remote -v. 执行时加上 -v 参数，你还可以看到每个别名的实际链接地址。

提取远程仓库
git fetch 从远程仓库下载新分支与数据
git pull 从远端仓库提取数据并尝试合并到当前分支

推送到远程仓库
git push [alias] [branch]

删除远程仓库
git remote rm [别名]
