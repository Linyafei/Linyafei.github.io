pod init
什么都不显示。
–silent

显示工具的版本。
–version

显示更多调试信息。
–verbose

显示不带ANSI代码的输出。
–no-ansi

显示指定命令的帮助横幅。
–help

pod install

pod repo update安装前强制运行。
–repo-update

禁止在安装过程中对Podfile或Podfile.lock进行任何更改。
–deployment

忽略项目缓存的内容，并强制进行完整的Pod安装。这仅适用于已启用增量安装的项目。
–clean-install

项目目录根目录的路径。
–project-directory=/project/dir/

pod update

从中更新依赖容器的源。多个来源必须以逗号分隔。
–sources=https://cdn.cocoapods.org/

更新期间要排除的豆荚。多个Pod必须以逗号分隔。
–exclude-pods=podName

忽略项目缓存的内容，并强制进行完整的Pod安装。这仅适用于已启用增量安装的项目。
–clean-install

项目目录根目录的路径。
–project-directory=/project/dir/

pod repo update在安装之前跳过运行。
–no-repo-update

pod outdated

项目目录根目录的路径。
–project-directory=/project/dir/

pod repo update在安装之前跳过运行。
–no-repo-update

从CocoaPods解集成项目。从您的Xcode项目中删除所有CocoaPods痕迹。
pod deintegrate

项目目录根目录的路径。
–project-directory=/project/dir/

显示窗格环境。
pod env

浏览
pod search

QUERY将正则表达式解释为。
–regex

仅按名称搜索。
–simple

显示其他统计信息（例如GitHub观察器和forks）。
–stats

在cocoapods.org上搜索。
–web

将搜索限制为iOS支持的Pod。
–ios

将搜索限制为macOS支持的Pod。
–osx

将搜索限制为watchOS支持的Pod。
–watchos

将搜索限制为tvOS支持的Pod。
–tvos

不要将搜索结果传递到寻呼机中。
–no-pager

pod list

–update

显示其他统计信息（例如GitHub观察器和forks）。
–stats

pod try
